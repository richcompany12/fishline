## 📌 Executive Summary (요약)

### 🎯 프로젝트 목표
FISHLINE의 **플로팅 카운터 기능만 추출**하여 국가별 맞춤 버전으로 글로벌 출시.
후진국·개발도상국일수록 **수동 카운팅 수요가 높다**는 시장 인사이트를 바탕으로 **블루오션 개척**.

### 🔑 핵심 차별화
```
🏆 경쟁자 대부분 "앱 안 카운터"만 제공
🏆 우리는 "다른 앱 위에 떠있는 플로팅 카운터" → 독보적!
🏆 멀티 아이템 지원 (여러 항목 동시 관리)
🏆 백그라운드 완벽 작동
```

### 📊 3년 목표 수익
- **1년차**: 월 100~300만원 (3~5개국)
- **2년차**: 월 500~1,000만원 (10개국)
- **3년차**: 월 1,500~3,000만원 (20개국+)

## 1. 시장 분석

### 🌍 글로벌 카운터 앱 시장 현황

| 시장 | 경쟁 강도 | 플로팅 기능 제공 | 진입 기회 |
|---|---|---|---|
| 🇰🇷 한국 | 중 | 거의 없음 | ⭐⭐⭐ |
| 🕌 중동 (20억 무슬림) | 저 | 허술함 | ⭐⭐⭐⭐⭐ |
| 🇯🇵 일본 | 중 | 없음 | ⭐⭐⭐⭐ |
| 🇻🇳 베트남 | 저 | 없음 | ⭐⭐⭐⭐⭐ |
| 🇹🇭 태국 | 저 | 없음 | ⭐⭐⭐⭐ |
| 🇮🇩 인도네시아 | 중 | 일부 있음 | ⭐⭐⭐ |
| 🇮🇳 인도 | 중 | 없음 | ⭐⭐⭐ |
| 🇷🇺 러시아 본토 | 저 | 없음 | ⭐⭐ (결제 제약) |
| 🇨🇳 중국 | 높음 | 일부 있음 | ⭐ (Play Store X) |

### 💡 핵심 인사이트
1. **플로팅 기능 보유 앱 = 거의 없음** → 대표님이 가진 Kotlin 네이티브 기술이 해자(moat)
2. **후진국일수록 수동 카운팅 수요 높음** (티켓팅, 재고, 검수 자동화 낮음)
3. **현지화된 디자인의 카운터 앱 부재** → 시각적 차별화 가능

## 2. 타겟 국가 전략

### 🎯 우선순위별 진출 로드맵

#### 🥇 1순위: 🕌 중동 (사우디/UAE/이집트)
```
타겟 사용자:
  - 디카르(기도 횟수) 카운팅 → 20억 무슬림
  - 극장/모스크 방문객 카운팅
  - 쇼핑몰 검표원
  
킬러 기능:
  - 디카르 프리셋 (수브한알라 33회 등)
  - 목표치 달성 시 진동/소리
  - Qibla 방향 표시 (보너스)
  
수익 모델:
  - 유료앱 $2~3 (구매력 높음)
  - 광고 제거 IAP $5
  
예상 월 수익: 100~300만원
```

#### 🥈 2순위: 🇻🇳 베트남
```
타겟 사용자:
  - 공장 품질 검수
  - 시장 노점 상인
  - 젊은 세대 (35세 미만 70%)
  
킬러 기능:
  - 매출 계산기 연동
  - TikTok 공유 버튼
  
수익 모델:
  - 광고 중심 (AdMob)
  - 낮은 단가, 높은 볼륨
  
예상 월 수익: 50~150만원
```

#### 🥉 3순위: 🇹🇭 태국
```
타겟 사용자:
  - 사원 참배자
  - 시장 상인
  - 관광 가이드
  
킬러 기능:
  - 불교 만트라 카운터
  - 태국어 UI
  - LINE 공유
  
예상 월 수익: 80~200만원
```

#### 🏅 4순위: 🇯🇵 일본
```
타겟 사용자:
  - 박물관/미술관 직원
  - 공연장 스태프
  - 도서관 이용자 카운트
  
킬러 기능:
  - 무음 모드 (정숙 중시)
  - 미니멀 디자인
  - 일본식 만년력 연동
  
수익 모델:
  - 유료앱 ¥200~400
  - 품질로 승부
```

#### 🎖 5순위: 🌍 러시아어 버전 (글로벌 러시아어권)
```
타겟: 카자흐스탄, 우즈베키스탄, 우크라이나, 이스라엘 러시아계, 독일 러시아계
이유: 러시아 본토는 결제 제약, but 러시아어권 5천만명 타겟 가능
배포: Google Play (러시아 본토 제외)
```

## 5. 신규 기능 상세

### 🎈 5-1. 플로팅 버튼 타입 선택 (⭐핵심 신규 기능)

#### 📌 **타입 A: 확장형 (목록별 버튼 전체 표시)**
```
화면 표시:
┌──────────────────┐
│ 🔵 쭈꾸미   42  │ ← 각 아이템마다 버튼
│ 🔴 갑오징어 17  │
│ 🟢 새우     03  │
└──────────────────┘

장점:
  ✅ 한 번에 여러 항목 카운트 가능
  ✅ 빠른 전환 (클릭 한 번으로 다른 항목 카운트)
  ✅ 현재 상태 한눈에 보임
  
단점:
  ⚠️ 화면 공간 많이 차지
  ⚠️ 다른 앱 사용에 방해 될 수 있음
  
사용 시나리오:
  - 낚시 (여러 어종 동시 카운트)
  - 검표원 (성인/어린이/VIP 동시)
  - 제조 품질 검수 (OK/NG/재검)
```

#### 📌 **타입 B: 컴팩트형 (현재 방식 - 단일 버튼 + 미니패널)**
```
화면 표시:
┌────┐
│ 🔵 │ ← 원형 버튼 하나만
└────┘
  ☰ 탭하면 미니패널 열림

장점:
  ✅ 화면 공간 최소화
  ✅ 이동이 편함
  ✅ 깔끔함
  
단점:
  ⚠️ 다른 항목 카운트하려면 패널 열어야 함
  
사용 시나리오:
  - 단순 카운트 (기도, 운동)
  - 주 1개 항목 + 가끔 다른 항목
```

#### 📌 **타입 선택 UI 설계**

설정 화면에 새 섹션 추가:

### 💾 5-3. 데이터 저장 기능 ⭐ (핵심 신규)

#### 🎯 기본 컨셉
**매일 카운트한 데이터를 자동 저장 + 메모 기능으로 추억 관리**

#### 📌 데이터 구조 설계

```typescript
interface CountRecord {
  id: string;              // 고유 ID
  date: string;            // 날짜 (20260418)
  memo: string;            // 사용자 메모 (ex: "무창포 브이호")
  items: CountItemRecord[]; // 카운트된 아이템들
  totalCount: number;      // 총 카운트
  createdAt: number;       // 생성 시간 (타임스탬프)
  location?: string;       // 위치 (선택)
}

interface CountItemRecord {
  name: string;            // 아이템명
  count: number;           // 카운트
  color?: string;          // 색상
}
```

#### 📌 자동 저장 로직

```
[저장 트리거]
1. 사용자가 "오늘 카운트 저장" 버튼 누르면
2. 자동 저장: 자정(24:00) 지나면 자동 저장
3. 앱 종료 시 자동 저장 (옵션)

[저장 형식]
날짜키: 20260418
메모: "무창포 브이호" (사용자 입력)
아이템: [쭈꾸미 42, 갑오징어 17, 새우 3]
```

### ✅ 안전한 차별화 (3-Layer)

#### 🎯 Layer 1: 앱 레벨
```
각 앱마다 반드시 차별화:
  ✅ 패키지명: com.richcompany.{countrycode}
     - 한국: com.richcompany.fishlineapp
     - 사우디: com.richcompany.dhikrcounter
     - 베트남: com.richcompany.smartcounter_vn
  ✅ 앱 이름: 완전히 다르게
  ✅ 아이콘: 베이스 유지하되 색상/심볼 변경
  ✅ 기본 언어: 현지어
```

#### 🎯 Layer 2: 스토어 리스팅
```
각 앱마다 완전히 다른 리스팅:
  ✅ 스크린샷: 현지 배경/모델로 재촬영 (8장+)
  ✅ 프로모 영상: 현지 내레이션
  ✅ 설명: 현지 사용 시나리오로 새로 작성
  ✅ 키워드: 현지 검색어 리서치 반영
  ✅ 카테고리: 가능하면 다르게 설정
```

#### 🎯 Layer 3: 기능 레벨 ⭐ 가장 중요!
```
각 버전에 1~2개 지역 특화 기능 추가:

🕌 중동: 디카르 프리셋 + Qibla

→ "같은 뼈대, 다른 심장" 전략
```
